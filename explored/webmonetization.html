Getting it Working

Ease of getting it to work:

3/4 out of 5

When we started the project only had hugo documentation which was a little strange, but now it‚Äôs supported as a plugin by Angular, Vue, Wordpress, Jekyll and new ones popping up every day.

High level steps from the web monetization quick start guide:

To monetize your site, you must:

  1. Set up a web monetized receiver (wallet) for receiving payments.
  2. Get your wallet's payment pointer.
  3. Create a special tag that tells Web Monetization providers how to pay you.
  4. Add the tag to each page of your website that you want to monetize

What trip ups did we have to keep an eye out for?

  • Anticipate setting up a wallet. You don't actually need to add your bank account. You will need to "add funds" and then generate a payment pointer which you then add to your meta tag. Great instructions for uphold here
  • Your site must be https to enable web monetization
  • When we initially did it there was only documentation for Hugo - if you got the Hugo route beware its a theme (not a plugin), and the second ‚Äúoption‚Äù for setting up a theme worked.
  • Ultimately on this site we implemented by adding the meta tag in the html head and then adding some custom css/js to get it working as advised on the web monetization site - they said this is not a good way of doing (seems very insecure) but did not offer a better solution ü§£
